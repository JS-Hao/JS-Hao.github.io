前端面试题不完全汇总

发表于 2016-09-19

校招大军已经在浩浩荡荡地进行中了，我也是参与校招的其中一份子（属于增大分母坐吃等死型人才），在面试过程中面试官不乏会提出各种各样的技术问题来难倒我们，往往很多应聘者就挂在了技术面这一难关。
为了能够顺利地通过面试，本人特地上网搜索罗列出了一些面试题以及答案（网上的答案不一定正确，因此每一个问题的答案我也做了一番思考以及稍微的改动，当然如果有一些不正确的地方，也欢迎大家指正出来~），仅供参考。当然，我们不能够抱着投机取巧的心态去对待面试（以为刷刷题就能够混过面试了？too young），对于下面所提的每一个问题，都值得我们认真地去思考。

  1. <!DOCTYPE>的作用是什么？有哪些模式？各自有什么特点？如何触发这些模式？区分它们有啥意义？
    • 作用 <!DOCTYPE>声明位于文档的最上方，用于告知浏览器以什么文档类型规范来解析这个文档。
    • 模式 三种：标准模式、准标准模式和混杂模式
    • 特点 标准模式会使浏览器的行为更接近标准行为，而混杂模式则是按浏览器自身的特性去解析文档（因此不同的浏览器之间的解析存在差异），而准标准模式则是大部分特性符合标准，而在处理图像间隙方面的时候还是存在差异
    • 触发 通过对<!DOCTYPE>的不同声明，可触发不同的模式，对于HTML5而言，可使用!DOCTYPE html声明
    • 区分 如果不区分（即DOCTYPE不存在或者格式不对），会导致文档以默认格式（混杂模式）解析
  2. 行内元素有哪些？块级元素有哪些？空（void）元素又有哪些？
    • 行内元素 a b span img strong em 以及行内块元素 input select img
    • 块级元素 div ul ol li dl dt dd h1 h2 h3 p等
    • 空元素 （即没有内容的元素，开始标签和结束标签之间的内容是元素的内容）br hr img input link meta
  3. CSS的盒子模型有哪些，有什么区别？
    • 标准W3C盒子模式 margin、border、padding、content，其中content不包括其他部分，height和width指的就是content的高与宽
    • 盒子模型 margin、content（其中包含了border和padding），height和width指的是包含了border和padding的content的高与宽，注意，如果我们没有给文档加上DOTYPE声明，则各浏览器会以自己的理解去解析文档，IE浏览器会把盒子解析成IE盒子
  4. link和@import的区别？
    • link属于XHTML标签，而@import是CSS提供的一种方式；
    • 加载的顺序。当一个页面被加载的时候，link引用的CSS会同时被加载，而@import引用的CSS会等到页面全部下载完之后再被加载；
    • 兼容性的差别。@import是CSS2.1提出的，所以老版本浏览器不支持，只有在IE5以上才能识别，而link无此问题；
    • 使用DOM去控制样式的差别，当使用JavaScript去控制DOM改变样式的时候，只能使用link标签，因为@import不受DOM的控制
    • 注意 @import要写在其他样式的开头前，否则对于部分浏览器而言它会被忽略
  5. b标签与em、strong标签的区别？

    b标签用于加粗字段，而em、strong用于强调文本，em可使字段变成斜体，strong可使字段加粗，且其强调程度大于em

  6. script标签放在head与body的区别？
    • 根据浏览器从上至下的加载顺序可以知道，如果把script标签放在head标签内，意味着浏览器会先去下载js文件，再去加载body内的内容，因此，如果js文件很多或者网速很卡的话，在等待js文件下载期间，网页呈现一片空白；而如果将它放置在底部，则js文件的下载会等到整个body内容加载好才进行，在js代码解析前用户便能够观看到整个网页内容，用户体验比前者好；
    • 如果把script标签放在head中同时又不将js代码写在window.onload或者$(document).ready事件内，若代码中含有控制DOM的操作也许会报错（还未解析到目标元素），而放在body底部则不需要将代码写进上述事件中，不会出现上述问题
  7. $(document).ready与window.onload的区别？
    • $(document).ready会在页面所有dom完全就绪的时候调用
    • window.onload会在页面包含图片等文件在内的所有元素都加载完成时调用
  8. CSS选择符有哪些？哪些属性可以继承？优先级算法如何计算？CSS3新增的伪类有哪些？
    • CSS选择符 ID选择器（#myid），类选择器（.classname），标签选择器（div），相邻选取器（div+p），子选择器（ul > li），通配符（*），属性选择器（a[rel = ‘external’]），伪类选择器（a:hover）
    • 可继承 font-sizen font-family color
    • 不可继承 border padding margin width height
    • 优先级 就近原则，样式定义最近者为准，载入样式以最后载入的定位为准，!important > 内联 > id > class > tag
    • 新增伪类 p:first-of-type p:last-of-type p:only-of-type p:nth-child(1) :enabled :disabled :checked等
  9. 如何居中一个div，如何居中一个浮动的元素？

    • 居中div 给div设置一个宽度，同时添加margin: 0 auto

    • 居中浮动元素 先为元素设置宽度，使其浮动后向右偏移50%的同时左外边距向左偏移其宽度的一半，注意，父元素应该改成相对定位或绝对定位

  10. 浏览器的内核分别是什么？经常遇到的浏览器的兼容性问题有哪些？解决方法是什么？常用的hack技巧有哪些
    • 内核 Trident（IE）、Gecko（Firefox）、Webkit（Safari、Chrome）、Presto（Opera）
    • 常见的浏览器兼容性问题、解决方法将以一个专题的形式在后续放出~
    • hack手法 关于hack，我的博客的其他文章已经提到：
      • CSS hack之CSS属性前缀法
      • CSS hack之选择器前缀法
  11. HTML5\CSS3有哪些新特性？移除了哪些元素？如何处理HTML5新标签的浏览器兼容性问题？如何区分HTML和HTML5？

    • HTML5新特性

      • 用于绘画的canvas元素
      • 用于媒体播放的audio元素和video元素
      • 本地离线存储:localStorage长期存储数据，浏览器关闭后数据不丢失，sessionStorage的数据在浏览器关闭后自动删除
      • 语义化更好的内容元素，如article、footer、header、nav、section
      • 新的表单元素：datalist、keygen、output
      • 新的input类型，如date、time、email、url、search
      • 新的技术：Web Worker、server-sent event、Geolocation

    • CSS3新特性 实现圆角、阴影、对文字加特效，增加了更多的CSS选择器，多背景以及RGBA

    • 移除的元素
      • 纯表现的元素：basefont、big、center、font、s、strike、tt、u
      • 对可用性产生负面影响的元素：frame、frameset、noframes

    • 旧浏览器兼容HTML5

      • 通过document.createElement的方法产生标签，这样可以让浏览器识别标签和CSS引擎知道它的存在（要放在head中，因为必须在元素解析前得知道它的存在，否则无效）

      • 使用html5shiv，这套成熟的框架可以让浏览器和引擎提前知道新标签的存在，同样也需要在head中引入
